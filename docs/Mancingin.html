Ringkasan

Apa yang dilakukan extension ini
  1. Kemampuan generik SQLite penuh. Membawa seluruh kemampuan generik dari komponen open-source org.bennedum.SQLite.SQLite (Execute, Select, Insert, Update, Delete, Replace, transaksi, dan variasi Async-nya).
  2. Hasil query konsisten. Selalu mengembalikan hasil query SELECT sebagai YailList berbentuk “List of Lists” (setiap baris = satu YailList), termasuk untuk hasil satu kolom, sehingga mudah di-parse langsung oleh komponen ListView melalui properti .Elements.
  3. Inisialisasi otomatis. Otomatis membuat & membuka berkas database mancing.db di storage internal aplikasi begitu komponen diletakkan di Screen — constructor memanggil OpenDatabase() secara otomatis, lalu menjalankan seluruh skema (tabel, index, view, trigger) saat database pertama kali dibuat.
  4. Blok level tinggi khusus domain. Menyediakan blok-blok fungsional khusus lomba mancing seperti TambahLombaEvent, DaftarPeserta, dan InputTimbanganIkan.

TambahLombaEvent(nama text, catatan text)returns list

Membuat event/lomba baru. Output: [[id_event, "Aktif"]].

DaftarPeserta(idLomba number, noLapak number, namaPeserta text, noHp text, joinC number, joinSS number)returns list

Mendaftarkan peserta baru ke suatu lapak pada lomba tertentu. Akan gagal (dan memicu event Error) jika melanggar UNIQUE(id_lomba, no_lapak). Output: [[id_peserta, "Terdaftar"]] atau [[-1, pesan_error]].

GantiSesi(idLomba number, namaSesiBaru text)returns list

Menutup sesi yang sedang berjalan (status ‘Mulai’ → ‘Selesai’) lalu membuka sesi baru berikutnya, dieksekusi dalam satu transaksi. Output: [[id_sesi_baru, "Mulai"]].

InputTimbanganIkan(idPeserta number, idSesi number, beratKg number)returns list

Mencatat hasil timbangan ikan seorang peserta pada suatu sesi. Ditolak jika berat bernilai negatif (CHECK constraint). Output: [[id_timbangan, "Tercatat"]] atau [[-1, pesan_error]].

CatatTransaksiKantin(idLomba number, idPeserta number, idMenu number, qty number, jumlah number, keterangan text)returns list

Mencatat transaksi pembelian kantin oleh peserta. Otomatis memicu Financial Engine Trigger untuk memperbarui kas utama & buku besar. Output: [[id_transaksi, "Tercatat"]] atau [[-1, pesan_error]].

AmbilDaftarLapakKosong(idLomba number)returns list

Melacak nomor lapak kolam yang belum terisi pada suatu lomba, memakai CTE Recursive berdasarkan setting jumlah_lapak. Output: [[1], [3], [4], [7], ...].

AmbilJurnalBukuKas(idLomba number)returns list

Mengambil seluruh rekam arus kas (buku besar) suatu lomba, terurut kronologis. Output: [[id_transaksi, jenis, kategori, jumlah, saldo_setelah, keterangan], ...].

AmbilKlasemenLiveGlobal(idLomba number)returns list

Membaca klasemen live keseluruhan (langsung dari view v_klasemen). Output: [[no_lapak, nama_peserta, total_berat_kg, jumlah_tangkapan], ...].

AmbilKlasemenPerSesi(idLomba number, idSesi number)returns list

Membaca klasemen per sesi/babak (langsung dari view v_klasemen_sesi). Output: [[no_lapak, nama_peserta, total_berat_kg_sesi, ikan_terberat_sesi], ...].

AmbilSkorDuelSampinganC(idLomba number, idSesi number)returns list

Mengevaluasi skor duel taruhan sampingan C antar lapak secara realtime (langsung dari view v_sampingan_c_live). Output: [[lapak_a, nama_a, total_a, lapak_b, nama_b, total_b, pemenang_duel], ...].

InisialisasiSkema()returns list

Menjalankan ulang seluruh DDL skema (tabel, index, view, trigger) secara manual. Biasanya tidak perlu dipanggil karena sudah otomatis saat database pertama kali dibuat. Output: [["Sukses", "Skema Terbentuk"]] atau [["Gagal", pesan_error]].
