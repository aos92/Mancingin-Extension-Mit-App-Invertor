TableGalatama

Menerima hasil query (YailList / List-of-Lists) dari MancinginExtension (Select, SelectSQL, AmbilKlasemenLiveGlobal, AmbilJurnalBukuKas, dst.) dan langsung menampilkannya sebagai tabel rapi — header, baris bernomor, warna selang-seling — di dalam sebuah TableArrangement yang sudah diletakkan di Designer, tanpa blok “for each” manual.

Ringkasan

Apa yang dilakukan extension ini
  • Jembatan query → tabel visual. TableGalatama menerima YailList (List-of-Lists) hasil query dari MancinginExtension dan menggambarnya langsung sebagai tabel di dalam TableArrangement yang sudah diletakkan di Designer.
  • Tanpa blok “for each” manual. Cukup panggil salah satu fungsi Tampilkan...(), oper TableArrangement tujuan dan hasil query — sisanya (header, nomor urut, format, warna) dikerjakan otomatis.
  • Format otomatis per jenis tabel. Setiap fungsi Tampilkan...() sudah tahu kolom mana yang berupa Rupiah, kg, atau tanggal, dan memformatnya sesuai jenis tabelnya masing-masing.
  • Fungsi universal. Untuk hasil query di luar 15 bentuk baku, gunakan TampilkanOtomatis() — cukup berikan judul kolom secara manual, tipe data (angka/teks) ditebak otomatis dari baris pertama.

Cara Pakai

Alur umum di App Inventor
  • Panggil MancinginExtension1.SelectSQL("SELECT ... ", buatDaftarKosong) (atau SelectSQLAsync + event AfterSelect) untuk mendapatkan YailList.
  • Panggil salah satu fungsi Tampilkan...() di bawah, oper TableArrangement tujuan dan YailList hasil query tadi.
  • TableGalatama otomatis: menghitung jumlah kolom & baris, menambah nomor urut, memformat Rupiah/kg/tanggal sesuai jenis tabelnya, memberi warna selang-seling, dan menampilkan pesan bila data kosong.
Catatan Penting
  • Setiap fungsi Tampilkan...() sudah mendokumentasikan query SQL yang disarankan — lihat bagian desc pada masing-masing method di bawah, lalu sesuaikan dengan skema database milikmu bila perlu.
  • Kolom & urutan data pada YailList yang dikirim harus sesuai urutan kolom yang diharapkan tiap fungsi (lihat daftar kolom pada deskripsi method). Jumlah kolom yang tidak cocok akan memicu event KesalahanData.
  • Gunakan TampilkanKlasemen bersama AmbilKlasemenLiveGlobal milik MancinginExtension — cukup buang 2 kolom pertamanya di blocks sebelum dioper ke sini.

Properties

10 properti
KlikBarisAktif boolean

Jika true, menyentuh satu baris data akan memicu event BarisDiklik. Default: false.

PesanDataKosong text

Pesan yang ditampilkan saat hasil query kosong (0 baris). Default: "Tidak ada data".

PrefixRupiah text

Awalan mata uang untuk kolom bertipe Rupiah. Default: "Rp".

UkuranTeksHeader number

Ukuran teks header, dalam sp. Default: 14.

UkuranTeksIsi number

Ukuran teks isi tabel, dalam sp. Default: 13.

WarnaBarisGanjil number (color)

Warna latar belakang baris data bernomor ganjil (1,3,5,...). Default: putih.

WarnaBarisGenap number (color)

Warna latar belakang baris data bernomor genap (2,4,6,...). Default: abu-biru muda.

WarnaHeader number (color)

Warna latar belakang baris header (ARGB). Default: biru.

WarnaTeks number (color)

Warna teks isi tabel (ARGB). Default: hitam keabu-abuan.

WarnaTeksHeader number (color)

Warna teks baris header (ARGB). Default: putih.

Events

3 event
BarisDiklik(namaTabel text, nomorBaris number, dataBaris list)

Dipicu saat satu baris data disentuh (hanya jika KlikBarisAktif = true). nomorBaris mulai dari 1, dataBaris = seluruh nilai asli baris tersebut (sebelum diformat).

KesalahanData(namaTabel text, pesan text)

Dipicu bila data yang diterima tidak sesuai (mis. jumlah kolom tidak cocok) atau terjadi kesalahan lain saat menggambar tabel.

TabelDitampilkan(namaTabel text, jumlahBaris number)

Dipicu setelah sebuah tabel selesai digambar. namaTabel = nama fungsi Tampilkan yang dipanggil (mis. "Klasemen"), jumlahBaris = banyaknya baris data yang ditampilkan.

Methods

16 method
Umum
BersihkanTabel(arrangement component)

Mengosongkan TableArrangement (dikembalikan menjadi 1 baris x 1 kolom kosong).

TampilkanOtomatis(arrangement component, judulKolom list, rows list)

Menampilkan hasil query apapun secara otomatis: judulKolom = daftar nama kolom (mis. buat daftar dengan blok “buat daftar” berisi teks), rows = hasil YailList dari MancinginExtension.SelectSQL/Select. Tipe data (angka rata kanan / teks rata kiri) ditebak otomatis dari nilai baris pertama.

Klasemen & Timbangan
TampilkanKlasemen(arrangement component, rows list)

TabelKlasemen: No Lapak, Nama Peserta, Total Berat (kg), Ikan Terberat (kg), Jumlah Tangkapan.
Saran SQL: SELECT no_lapak, nama_peserta, total_berat_kg, ikan_terberat, jumlah_tangkapan FROM v_klasemen WHERE id_lomba=? ORDER BY total_berat_kg DESC (atau pakai AmbilKlasemenLiveGlobal lalu buang 2 kolom pertamanya di blocks).

TampilkanLogTimbangan(arrangement component, rows list)

TabelLogTimbangan: No Lapak, Nama Peserta, Sesi, Berat (kg), Waktu.
Saran SQL (lewat MancinginExtension.SelectSQL): SELECT p.no_lapak, p.nama_peserta, s.nama_sesi, t.berat_kg, t.timestamp FROM timbangan t JOIN peserta p ON p.id=t.id_peserta JOIN sesi s ON s.id=t.id_sesi WHERE p.id_lomba=? ORDER BY t.timestamp DESC

Registrasi & Wilayah
TampilkanRegistrasi(arrangement component, rows list)

TabelRegistrasi: No Lapak, Nama Peserta, No HP, Join C, Join SS.
Saran SQL: SELECT no_lapak, nama_peserta, no_hp, CASE WHEN join_c=1 THEN 'Ya' ELSE 'Tidak' END, CASE WHEN join_ss=1 THEN 'Ya' ELSE 'Tidak' END FROM peserta WHERE id_lomba=? ORDER BY no_lapak

TampilkanWilayahKiri(arrangement component, rows list)

TabelWilayahKiri: No Lapak, Nama Peserta, No HP, Ikut C, Ikut SS (lapak ganjil).
Saran SQL: SELECT no_lapak, nama_peserta, no_hp, CASE WHEN join_c=1 THEN 'Ya' ELSE 'Tidak' END, CASE WHEN join_ss=1 THEN 'Ya' ELSE 'Tidak' END FROM peserta WHERE id_lomba=? AND no_lapak % 2 = 1 ORDER BY no_lapak

TampilkanWilayahKanan(arrangement component, rows list)

TabelWilayahKanan: No Lapak, Nama Peserta, No HP, Ikut C, Ikut SS (lapak genap).
Saran SQL sama seperti WilayahKiri, ganti kondisi menjadi: no_lapak % 2 = 0

Taruhan (Sampingan)
TampilkanTaruhanC(arrangement component, rows list)

TabelTaruhanC: Sesi, Lapak A, Peserta A, Lapak B, Peserta B, Keterangan.
Saran SQL: SELECT s.nama_sesi, p1.no_lapak, p1.nama_peserta, p2.no_lapak, p2.nama_peserta, sc.keterangan FROM sampingan_c sc JOIN sesi s ON s.id=sc.id_sesi JOIN peserta p1 ON p1.id=sc.id_peserta1 JOIN peserta p2 ON p2.id=sc.id_peserta2 WHERE sc.id_lomba=? ORDER BY sc.id

TampilkanTaruhanSS(arrangement component, rows list)

TabelTaruhanSS: Sesi, Kategori, Pemenang, Hadiah (Rp).
Saran SQL: SELECT s.nama_sesi, ss.nama_kategori, COALESCE(p.nama_peserta,'-'), ss.hadiah_rp FROM sampingan_ss ss JOIN sesi s ON s.id=ss.id_sesi LEFT JOIN peserta p ON p.id=ss.id_peserta_pemenang WHERE ss.id_lomba=? ORDER BY ss.id

Riwayat / History
TampilkanHistoriBabak(arrangement component, rows list)

TabelHistoriBabak: Nama Sesi/Babak, Status.
Saran SQL: SELECT nama_sesi, status_sesi FROM sesi WHERE id_lomba=? ORDER BY id

TampilkanHistoriLomba(arrangement component, rows list)

TabelHistoriLomba: Nama Event, Mulai, Selesai, Status.
Saran SQL: SELECT nama_event, tanggal_mulai, tanggal_selesai, status FROM lomba_event ORDER BY id DESC

TampilkanHistoriPendaftaran(arrangement component, rows list)

TabelHistoriPendaftaran: Peserta, Biaya (Rp), Keterangan, Waktu.
Saran SQL: SELECT p.nama_peserta, tr.jumlah, tr.keterangan, tr.timestamp FROM transaksi tr JOIN peserta p ON p.id=tr.id_peserta WHERE tr.id_lomba=? AND tr.kategori='Pendaftaran' ORDER BY tr.timestamp DESC

TampilkanHistoriPenjualan(arrangement component, rows list)

TabelHistoriPenjualan: Peserta, Menu, Qty, Total (Rp), Waktu.
Saran SQL: SELECT p.nama_peserta, m.nama_menu, tr.qty, tr.jumlah, tr.timestamp FROM transaksi tr JOIN menu_food m ON m.id=tr.id_menu LEFT JOIN peserta p ON p.id=tr.id_peserta WHERE tr.id_lomba=? AND tr.kategori='Kantin' ORDER BY tr.timestamp DESC

TampilkanHistoriOperasional(arrangement component, rows list)

TabelHistoriOperasional: Jenis Operasional, Nominal (Rp), Keterangan, Waktu.
Saran SQL: SELECT sub_kategori, jumlah, keterangan, timestamp FROM transaksi WHERE id_lomba=? AND kategori='Operasional' ORDER BY timestamp DESC

TampilkanHistoriHadiah(arrangement component, rows list)

TabelHistoriHadiah: Penerima, Kategori Hadiah, Nominal (Rp), Keterangan, Waktu.
Saran SQL: SELECT COALESCE(p.nama_peserta,'-'), tr.sub_kategori, tr.jumlah, tr.keterangan, tr.timestamp FROM transaksi tr LEFT JOIN peserta p ON p.id=tr.id_peserta WHERE tr.id_lomba=? AND tr.kategori='Hadiah' ORDER BY tr.timestamp DESC

TampilkanHistoriKas(arrangement component, rows list)

TabelHistoriKas: Jenis, Kategori, Jumlah (Rp), Saldo Setelah (Rp), Keterangan, Waktu.
Saran SQL: SELECT jenis, kategori, jumlah, saldo_setelah, keterangan, timestamp FROM buku_kas WHERE id_lomba=? ORDER BY timestamp DESC (atau pakai AmbilJurnalBukuKas lalu sesuaikan kolomnya).
